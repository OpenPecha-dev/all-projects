  41. NT — 11/29/2021

    The notes will need a custom layout analysis model. The image quality definitely makes a big difference, as can be seem on Elie's images analysed by Google.
  42. [6:53 PM]
    To answer your question @phugon, the notes should be added in their own fields. Adding them to the base text is going to confuse the recognition model since you will give it an image of the line with the text of line+notes. I would suggest to run Transkribus and get the lines, then add the notes in simple square boxes and/or straight baselines. Drawing the contour is a bit of a waste of time.
  43. [6:57 PM]
    The HTR model is only trained on what is included in line cut out from the image and the text. So Transkribus' line segmentation ignoring the notes is fine and won't interfere with the training. Finding the notes is purely a layout analysis problem, once the boxes containing the notes are traced the note line+text pairs can be added to the training.

  44. NT — 11/29/2021

    In the short term I would do the following: TRAINING DATA CREATION - layout analysis of main text with Transkribus - layout cleanup with the script - add main text through manual input or with a script for existing input - manually trace baselines or simple boxes for notes - add text through manual input or with a script for existing input - train the recognition model on both base text and notes RECOGNITION - layout analysis of the main text with Transkribus - layout cleanup with the script - manually trace baselines or simple boxes for notes - HTR - post-processing script to reconstruct lines (with customization for notes)

  45. NT — 11/29/2021

    In the long term I would look into training a model for pecha layout analysis that includes notes. The more training images with baseline annotations for both text and notes, the earlier this will be possible.
